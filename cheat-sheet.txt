Git global setup:

    git config --global user.name "Ralf Hinze"
    git config --global user.email "ralf.hinze@cs.uni-kl.de"


Create a new repository (you only do this once):

    git clone git@pl-git.informatik.uni-kl.de:hinze/runtime-bounds.git
    cd GP-guru
    touch README.md
    git add README.md
    git commit -m "add README"
    git push -u origin master

Add a new file locally:

    git add <filename>

Commit changes locally:

    git add -u ; git commit

Pull changes from repository (merges automagically):

    git pull

Push local changes to repository:

    git push

Push local changes to repository (first time):

    git push -u origin master

Rename file:

    git mv <source> <dest>

Tagging:

    - lists tags
      git tag
    - create tag
      git tag -a 1.4 -m "my version 1.4"
    - show tag data
      git show 1.4
    - share tags
      git push --tags
    - check out tag
      git checkout -b draft 1.4

Inspect logs (*):

    git log
    git history

Inspect changes (<hash value> from git history):

    git diff
    git diff <hash value> <hash value>

Go back and forth in history (<hash value> from git history):

    git checkout <hash value>
    git checkout master

See also
    https://git-scm.com/book/

(*) The history command is an alias and requires the following
definition in your ~/.gitconfig file:
[alias]
        history = log --pretty=format:\"%h %ad | %s%d [%an]\" --graph --date=short
